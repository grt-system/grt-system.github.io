Relatório do Projeto de Microsserviços

1. Visão Geral

Este documento apresenta uma visão detalhada da arquitetura de microsserviços implementada em nosso sistema.

2. Arquitetura

  • Gateway API
  • Serviço de Autenticação
  • Serviço de Usuários
  • Serviço de Produtos
  • Serviço de Pedidos

3. Tecnologias Utilizadas

  • Docker para containerização
  • Kubernetes para orquestração
  • Spring Boot para desenvolvimento dos serviços
  • MongoDB e PostgreSQL para persistência
  • RabbitMQ para mensageria

4. Benefícios Alcançados

  • Alta disponibilidade
  • Escalabilidade independente
  • Facilidade de manutenção
  • Melhor isolamento de falhas

5. Desafios e Soluções

  • Consistência de dados: Implementação de Saga Pattern
  • Monitoramento: Utilização de ELK Stack
  • Rastreamento: Implementação de Distributed Tracing

6. Próximos Passos

  • Implementação de Circuit Breaker
  • Melhoria na observabilidade
  • Automatização de testes end-to-end
